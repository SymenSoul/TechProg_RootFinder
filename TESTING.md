# Руководство по работе с юнит-тестами (MathClientTests & MathServerTests)

Данный документ описывает процессы запуска, анализа результатов и написания новых юнит-тестов для клиентской и серверной частей приложения. Тесты построены на стандартной библиотеке Qt `QTest` (модуль `testlib`).

---

## 1. Запуск через графическую среду Qt Creator

Этот способ наиболее удобен для повседневной разработки и демонстрации работы проекта.

1.  **Открытие проектов тестов**:
    *   В Qt Creator выберите `File -> Open File or Project...` (Файл -> Открыть файл или проект...).
    *   Перейдите в директорию `MathClientTests` и откройте файл `MathClientTests.pro`.
    *   Перейдите в директорию `MathServerTests` и откройте файл `MathServerTests.pro`.
2.  **Сборка и запуск**:
    *   В левой панели Qt Creator выберите необходимый проект сборки (переключитесь с основного приложения на `MathClientTests` или `MathServerTests`).
    *   Нажмите зеленую кнопку запуска (зеленый треугольник на левой панели) или используйте сочетание клавиш `Cmd+R` (на macOS) / `Ctrl+R` (на Windows/Linux).
3.  **Просмотр результатов**:
    *   Текстовый лог работы тестов отобразится во вкладке **Application Output** (Вывод приложения) на нижней панели.
    *   Qt Creator автоматически парсит вывод `QTest` и отображает результаты проверок в виде древовидного графического списка во вкладке **Test Results** (Результаты тестов) в нижней части экрана.

---

## 2. Запуск через консоль (Терминал)

Сборка и запуск тестов без графической оболочки.

### Тесты клиента (`MathClientTests`):
```bash
cd MathClientTests
qmake MathClientTests.pro
make
./MathClientTests
```

### Тесты сервера (`MathServerTests`):
```bash
cd MathServerTests
qmake MathServerTests.pro
make
./MathServerTests
```

### Интерпретация вывода в консоли:
*   Строка `PASS` означает, что тестовый сценарий успешно выполнен и все утверждения подтвердились.
*   Строка `FAIL!` указывает на то, что проверка не пройдена. В выводе будет указано имя тестовой функции, конкретный файл и номер строки кода, на которой произошел сбой, а также ожидаемое и фактическое значения.
*   Блок `Totals` в самом конце показывает статистику: общее число выполненных проверок (`passed`), упавших тестов (`failed`) и пропущенных (`skipped`).

## 3. Добавление новых тестов

Чтобы добавить новый тест в любой из проектов, выполните следующие шаги:

1.  **Объявление метода**:
    Откройте файл `main.cpp` соответствующего проекта тестов и добавьте имя вашей тестовой функции в раздел `private slots` класса тестов:
    ```cpp
    private slots:
        void testMyNewFeature();
    ```
    *Важно*: `QTest` автоматически распознает и запускает в качестве тестов только те методы, которые объявлены именно в секции `private slots`.

2.  **Реализация логики**:
    Реализуйте тело метода в конце файла `main.cpp`:
    ```cpp
    void ClientTests::testMyNewFeature()
    {
        // Вызов тестируемого кода
        QString result = ClientController::getInstance()->parseServerResponse("success");
        
        // Сравнение результата с ожидаемым значением
        QCOMPARE(result, QString("SUCCESS"));
    }
    ```

3.  **Полезные макросы для проверок**:
    *   `QCOMPARE(actual, expected)`: проверяет равенство двух значений. Если значения не равны, тест завершается неудачей.
    *   `QVERIFY(condition)`: проверяет, истинно ли логическое выражение `condition`. Принимает `true` или `false`.
    *   `QVERIFY2(condition, message)`: аналогично `QVERIFY`, но позволяет вывести текстовое сообщение `message` в лог при падении теста.

4.  **Пересборка**:
    После внесения изменений выполните повторную сборку и запуск проекта.
